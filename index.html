CI/CD PIPELINE

Project Overview:

This project demostrates how infrastructure as code(IaC), container orchestration, and CI/CD automation can be combined to deliver a seamless deployment process:
- Terraform provisions AWS infrastructure, which includes ECS clusters, task definitions, and supporting resources
- AWS ECS (Fargate) runs the Nginx container without requiring server management
- Jenkins is then intergrated to handle upadates. Whenever a new Nginx version is needed, Jenkins updates the ECS task definition through Terraform and redeploys the container automatically.
The result is a clean workflow : Terraform builds the foundation, Jenkins drives the ongoing updates. Each time Jenkins runs, the ECS service redeploys with new Nginx image, and the website confirms success with the classic "Welome to Nginx"

Steps taken to complets this project :

1. Provision Infrastructure with Terraform

Install Terraform and configure AWS credentials
Define resources in Terraform configuration files
Initialize Terraform and apply the changes
Verify the infrastructure is created successfully


2. Set Up Jenkins

- Installed Jenkins on Docker
- Configured Jenkins with required plugins
- Stored AWS credentials securely in Jenkins using Credentials Manager


3. Create Jenkins Pipeline

This pipeline:
- Runs the build step and Deploys to AWS
- Updates the ECS task definition through Terraform and redeploys the container automatically


4. Validate Deployment

- Accessed the deployed website and confirmed it is functioning correctly.
- Verified the Nginx version matches the updated image.
